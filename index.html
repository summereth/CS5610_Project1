RookieDB: Database Management System Implementation (W.I.P.)
RookieDB is a comprehensive database management system implementation that supports executing transactions in series. The project implements core database functionalities including B+ tree indices for efficient data retrieval, query optimization algorithms, and a robust transaction management system with multigranularity locking for concurrent execution.
The system features a modular architecture with components for buffer management, disk space management, and query processing. Implementation includes sophisticated memory management through buffer pools, write-back caching mechanisms, and efficient page directory structures for heap file management, all while maintaining ACID properties for transactions.
Key Technical Features:
Core Components: B+ tree indices, query optimizer, buffer manager, disk space manager
Transaction Management: Multigranularity locking, ACID compliance
Memory Management: Write-back buffer cache, LRU/Clock eviction policies
Query Processing: Complex operator implementation, cost-based optimization
Architecture: Command-line interface, SQL parser, table/record management

Reindeers Forever Gifts: Donation Platform for Non-Profit Org
Reindeers Forever Gifts is a full-stack donation platform built for a non-profit organization, featuring a microservices architecture that handles secure donations and user management. The platform combines a React-based frontend with a Java Spring Boot backend to deliver a seamless donation experience while ensuring robust security through AWS Cognito authentication.
The system's architecture emphasizes scalability and maintainability through RESTful APIs and MySQL database integration, achieving 85%+ test coverage through comprehensive unit and integration testing. The frontend utilizes React Router for navigation and Context API for state management, creating an intuitive single-page application experience.
Key Technical Features:
Backend: Java, Spring Boot, MySQL, RESTful APIs
Frontend: React, React Router, Context API
Security: AWS Cognito authentication
Testing: JUnit for unit/integration tests, Postman for E2E testing

Gitlet: Mini Version-Control System Inspired by Git
Gitlet is a lightweight version-control system implemented in Java that mirrors core Git functionalities. This project implements essential version control operations including commit, checkout, merge, and reset, providing developers with a streamlined way to track and manage file changes.
The system utilizes SHA-1 hashing for content addressability and employs sophisticated data structures for efficient version management. By implementing tree and graph structures, Gitlet optimizes storage and retrieval of file versions while ensuring reliable commit history tracking and branch operations.
Key Technical Features:
Core Features: commit, checkout, merge, reset functionalities
Data Structures: Tree and graph implementations for version management
Storage: Java serialization for persistent data storage
Algorithms: Graph traversal for commit history and branch management
Security: SHA-1 hashing for content verification and addressing
